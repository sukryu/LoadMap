Go 백엔드 애플리케이션이 성공적으로 설치되었습니다!

------------------------------------------------------------
1. 배포된 리소스 확인
------------------------------------------------------------
아래 명령어를 사용하여 배포된 Pod, Service, Ingress(설정한 경우) 등의 상태를 확인할 수 있습니다.

   # Pod 상태 확인
   kubectl get pods -n {{ .Release.Namespace }}

   # Service 상태 확인 (외부 IP가 할당되었는지 확인)
   kubectl get svc -n {{ .Release.Namespace }}

   # Ingress 상태 확인 (Ingress가 활성화된 경우)
   kubectl get ingress -n {{ .Release.Namespace }}

------------------------------------------------------------
2. 애플리케이션 접근 방법
------------------------------------------------------------
[서비스 유형에 따른 안내]

- **LoadBalancer 타입 Service 사용 시:**
  외부 IP가 할당되면, 브라우저 또는 API 클라이언트를 통해 아래와 같이 접근할 수 있습니다.
  
     http://<외부_IP>:80
     또는 HTTPS를 사용하는 경우:
     https://<외부_IP>:443

- **Ingress 사용 시:**
  Ingress 설정에 따라 도메인 이름(예: api.example.com)을 통해 접근할 수 있습니다.
  
     예: https://api.example.com

------------------------------------------------------------
3. 추가 확인 사항
------------------------------------------------------------
- 애플리케이션 로그를 확인하여 정상 동작 여부를 확인합니다.
  
     kubectl logs -l app={{ include "go-api.labels" . }} -n {{ .Release.Namespace }}

- 배포 후, HPA(HorizontalPodAutoscaler)가 활성화된 경우 아래 명령어로 스케일링 상태를 확인합니다.
  
     kubectl get hpa -n {{ .Release.Namespace }}

- 필요에 따라 추가 환경 변수나 구성 변경이 있다면, values 파일을 수정한 후 차트를 업그레이드하세요.
