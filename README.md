# 장기 개발자 로드맵 (군 복무 포함 feat. GPT O1 pro mode)

## 전체 개요

* Phase 0 (입대 전 3개월)
    * 학습 기반 마련 + 필수 자료 준비

* Phase 1 (군 복무 중 1년 6개월)
    * 이론 위주 학습 + 소규모 문제 풀이 (코딩 환경 제약이 있으므로 주로 CS 개념/알고리즘 이론 복습)

* Phase 2 (전역 후 복학 전 5개월)
    * 프로젝트/실습 집중 + 포트폴리오 구성

* Phase 3 (복학 이후 ~ 졸업까지 1 ~ 2년)
    * 심화/확장, MLOps 준비, 취업 준비비

## Phase 0. 입대 전 3개월
* 목표
    * 군 복무 중에 활용할 이론 자료, 노트, 플래시카드, 전자책 등 준비
    * 기본 CS 지식과 알고리즘 개념을 빠르게 점검("선수학습")
    * 짧은 코딩 실습으로 감각 유지

* 상세 계획
    1. 자료 준비
        * CS 기초(자료구조, 알고리즘, OS, 네트워크, DB) 관련 전자책/PDF/노트 요약
        * 알고리즘 노트: 각 알고리즘(정렬, 탐색, BFS/DFS, DP, 그리디, MST, 최단 경로 등)에 대한 핵심 아이디어와 시간 복잡도, 예시 코드(간단)
        * 학습 툴: Anki(또는 Quizelet, RemNote)등 간격 반복(Spaced Repetition) 앱에 기본 개념/공식/키워드 등록

    2. CS 이론/알고리즘 빠른 복습
        * 자료구조/알고리즘: 백준/프로그래머스 '기초 100문제' 식으로 선별된 단순 문제 풀기
        * 컴퓨터 네트워크/운영체제/DB: 인강 또는 유튜브 단기 정리 영상 시청 + 노트 정리
        * 미적분/선형대수 등 수학 기초: AI나 고급 알고리즘 시에 필요

## Phase 1. 군 복무 중 (1년 6개월)
* 목표
    * 코딩 환경 제약: 이병 ~ 일병 초반에는 싸지방 이용이 제한될 수 있으므로, **이론 학습 중심**
    * 스마트폰/전자책/종이책을 활용하여 CS/알고리즘/수학 개념을 반복적으로 학습
    * 싸지방 가능해지면, 짧게라도 온라인 문제 풀이(백준 등) 시도

* 상세 계획
    1. 학습 방식
        * 이론 복습/정리
            - 매일 30분 ~ 1시간: 알고리즘 개념, OS/네트워크, DB 개념을 노트나 플래시카드로 회독
            - Spaced Repetition(Anki 등)으로 키워드/공식 반복
        * 문제 풀이(싸지방 접근 가능 시)
            - 주 1 ~ 2회 백준(또는 프로그래머스)에서 단순 문제 풀기
            - PC 사용이 제한되면, 의사코드나 로직 구성만 종이에 써보기

    2. 학습 주제
        * CS 기초(자료구조/알고리즘) 재복습
        * OS(프로세스/스레드, 메모리, 스케줄링), 네트워크(TCP/IP, HTTP), DB(정규화, 트랜잭션) 개념 체화
        * 선형대수/통계 기초(일반 교양수준): 벡터/행렬 곱, 확률분포, 기댓값-분산
    
    3. 노트/기록 방법
        * TIL(Today I Learned) or 주간 정리: 지식이 누적되도록 1주에 한 번씩은 핵심 포인트를 요약
        * "DP 점화식 예시", "네트워크 계층별 프로토콜 특징"등 소단원별로 A4 한 장 요약

## Phase 2. 전역 후 (복학 전 5개월)
* 목표
    * 이론을 실제 프로젝트에 적용하여 백엔드/클라우드 개발 역량 및 포트폴리오 구축
    * C++ / Python / Go를 적극 활용하여 프로젝트 하나 이상 완성
    * 알바 병형: 시간을 효율적으로 분배(하루 4 ~ 5시간 정도 학습/개발 가능하도록 스케줄링)

* 상세 계획
    1. 프로젝트 집중
        - 백엔드 + 클라우드 실습
            - Django/FastAPI or Go(Gin/Echo)로 API 서버 개발
            - Docker 컨테이너, Kubernetes (minikube 등 로컬 환경) 배포
            - CI/CD 구현(Jenkins, Github Actions) -> 배포 자동화

            - PostgreSQL/MySQL 연동, ORM(SQLAlchemy, GORM 등) 사용

        - MLOps 작은 맛보기
            - 간단한 ML 모델(예: Scikit-learn) + 모델 서빙(Flask API)
            - Docker 이미지로 모델 서빙 -> 클라우드(혹은 로컬 K8s)에 배포

    2. 알고리즘/코딩 테스트 스킬 재점검
        - 매일/격일로 백준/프로그래머스 중급 문제 풀이
        - 이전에 군대에서 만든 알고리즘 노트를 기반으로 빠른 복습

    3. 지식 보완
        - 운영체제 심화(프로세스 관리, 메모리 가상화, 파일시스템 구조 등)
        - 네트워크 심화(TLS/HTTPS, HTTP2, WebSocket, gRPC)
        - 클라우드 서비스(AWS/GCP) Hands-on. (EC2/RDS/S3, IAM 보안 등)

    4. 포트폴리오 준비

## 복습 / 학습 방법 전반

1. Spaced Repetition
    - 군 복무 중, 전역 후, 복학 후에도 계속 쓸 수 있는 무기
    - 개념/공식/용러를 플래시카드로 만들어, 주기별(1일 후, 3일 후, 1주 후...) 자동 복습

2. TIL(Today I Learned) / 주간 회고
    - 개발/학습 중 새로 알게 된 것, 삽질한 것, 해결책 등을 짧게라도 문서화
    - 한 달 뒤 돌아봐도 어떤 걸 배웠는지 트래킹 가능

3. 프로젝트 + 이론 연계
    - 단순히 이론을 외우는 게 아니라, 프로젝트에서 "왜 이 DB 쿼리가 느린가?" -> "인덱스 구조, 트랜잭션 고립 수준"으로 연결
    - MLOps 쪽에서도 "배포/모니터링" 문제 -> "K8s의 Deployment/Service,  로깅/모니터링 시스템 이해"로 이어짐.

4. 디버깅과 에러 다이어리
    - 특정 에러 로그나 배포 실패 이슈가 발생할 때마다, 원인과 해결 과정을 정리 (이건 협업에서도 유용).

# 1. 컴퓨터 사이언스(CS) 기본기 다지기
## 1.1 데이터 구조와 알고리즘
### A. 기본 자료구조
- 배열(O), 연결 리스트(O), 스택(O), 큐(O)
- 트리(O), 그래프, 해시 테이블(O), 힙(O)
- Trie(트라이), 라디슐 트리(Radix Tree)
  - 문자열 검색 최적화, 자동완성 기능 구현
  - Patricia Trie 변형 구조
- Disjoint Set(Union-Find) 심화
  - Path Compression, Union by Rank 최적화
  - 동적 그래프 연결성, 군집화 응용

### B. 균형 트리 구조
- Red-Black Tree(O), AVL Tree(O)
- B-Tree, B+ Tree
  - 데이터베이스 인덱싱 원리
  - 디스크 기반 자료구조 이해

### C. 알고리즘 기초
1. 기본/고급 정렬
   - 선택(O), 삽입(O), 버블(O), 퀵(O), 머지(O), 힙 정렬(O)
   - 기수 정렬, 셸 정렬(O)
   - 하이브리드 정렬(Timsort 등)

2. 탐색 알고리즘
   - 선형/이진 탐색(O)
   - KMP, 라빈-카프, 보이어-무어
   - Z 알고리즘, Aho-Corasick

### D. 고급 그래프 알고리즘
1. 최단 경로
   - 다익스트라, 벨만-포드, 플로이드-워셜
   - Johnson 알고리즘 (음수 가중치)

2. 최소 스패닝 트리(MST)
   - 크루스칼, 프림 알고리즘
   - Borůvka 알고리즘

3. 네트워크 플로우
   - 최대 유량: 포드-풀커슨, 에드몬드-카프, 디닉
   - 최소 컷, 이분 매칭
   - Push-Relabel 알고리즘

4. 그래프 분석
   - 강연결 요소(SCC): Kosaraju, Tarjan
   - 단절점과 브리지 탐색
   - 2-SAT 문제 해결

### E. 고급 자료구조
1. 세그먼트 트리
   - 구간 쿼리(합, 최대/최소)
   - Lazy Propagation
   - Persistent Segment Tree

2. 펜윅 트리(Binary Indexed Tree)
   - 구간 업데이트 최적화
   - 2D 펜윅 트리

3. 문자열 자료구조
   - 접미사 배열(Suffix Array)
   - LCP 배열
   - 접미사 트리(Suffix Tree)
   - Manacher 알고리즘

### F. 동적 프로그래밍 심화
1. 최적화 테크닉
   - Knuth Optimization
   - Convex Hull Trick
   - Divide & Conquer DP

2. 특수 DP
   - 비트마스크 DP
   - Tree DP
   - Probability DP

### G. 수학/정수론 알고리즘
1. 정수론
   - 확장 유클리드
   - 중국인의 나머지 정리(CRT)
   - 밀러-라빈 소수 판별
   - 폴라드 로 소인수분해

2. 조합론
   - 번사이드 보조정리
   - 뤼카 정리
   - 포함-배제 원리

### H. 계산 기하학
1. 기본 알고리즘
   - Convex Hull (Graham Scan, Andrew)
   - Line Sweep
   - 점 내부 판정

2. 고급 주제
   - Voronoi Diagram
   - Delaunay Triangulation
   - KD-Tree, R-Tree

### 1.2 운영체제와 컴퓨터 네트워크
- **운영체제**:
  - 프로세스/스레드 관리, 메모리 관리, 파일 시스템, 동기화 기법
- **컴퓨터 네트워크**:
  - OSI 7계층, TCP/IP, HTTP/HTTPS 프로토콜, 소켓 프로그래밍.

### 1.3 데이터베이스 기초
- **관계형 데이터베이스**:
  - SQL, 정규화, 트랜잭션, 인덱스 등.
- **비관계형 데이터베이스**:
  - NoSQL 개념, MongoDB, Redis 등.

# 단어 정리

* ACPI는 UEFI와 "같은 기구(UEFI 포럼)"에서 관리되지만, 독립된 표준이다.
* UEFI는 "부팅 및 펌웨어" 인터페이스를 담당, ACPI는 "운영체제에서 하드웨어의 전력/열/성능 제어"를 담당.
* 리눅스 커널의 ACPI 관련 코드나 문서는, ACPI 스펙(UEFI.org에서 배포)을 기준으로 구현된 것이라고 보면 된다.
* 그러나 "ACPI = UEFI 내에 포함된 기능"으로 단정 짓기보다는, "UEFI와 ACPI가 모두 UEFI 포럼에서 관리되는 서로 연관된(그러나 별개의) 스펙"이라 보는 편이 정확하다.
